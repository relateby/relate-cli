# Implementation Plan: Packaging and Distribution

**Branch**: `005-packaging-distribution` | **Date**: 2026-05-11 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/005-packaging-distribution/spec.md`

## Summary

Automate the full release pipeline for `relate` so that pushing a version tag publishes prebuilt binaries to five platforms via GitHub Releases, Homebrew tap, npm (wrapper + platform packages), and crates.io — all within a single CI run. A project website at `cli.relateby.dev` serves OS-aware install instructions and stays in sync with each release.

Primary tool: **cargo-dist** for binary cross-compilation, install script generation, and Homebrew tap publishing. The npm distribution uses an `optionalDependencies` shim pattern (no postinstall scripts). The website is vanilla HTML/CSS/JS deployed to GitHub Pages from the `gh-pages` branch.

## Technical Context

**Language/Version**: Rust 1.85.0 (binary); JavaScript (Node.js shim for npm wrapper); HTML/CSS (website)
**Primary Dependencies**: cargo-dist (release automation); GitHub Actions; npm; Homebrew
**Storage**: N/A — no application data; distribution artifacts stored in GitHub Releases and npm registry
**Testing**: `cargo test` (existing); install script tested on fresh CI runners per platform; website validated manually
**Target Platform**: GitHub Actions runners (ubuntu-latest, macos-latest, windows-latest) for builds; GitHub Pages for website
**Project Type**: CLI distribution pipeline + static website (no changes to the `relate` binary itself)
**Performance Goals**: Full pipeline completes in ≤15 min; install via `curl | bash` completes in ≤60s on broadband
**Constraints**: No Apple Developer account (no macOS signing); no Authenticode cert (no Windows signing); single binary, no daemon
**Scale/Scope**: 5 platforms × 1 binary per release; 6 npm packages per release; 1 Homebrew formula per stable release

## Implementation Phases

### Phase 1 — cargo-dist bootstrap and GitHub Release pipeline

**Goal**: A `vX.Y.Z` tag on `main` produces a GitHub Release with 5 platform archives, checksums, and install scripts. No other channels yet.

**Tasks**:
1. Run `cargo install cargo-dist && cargo dist init` — select shell, powershell, homebrew, npm installers; tap = `relateby/homebrew-tap`; targets = 5 platforms. Commit `Cargo.toml` changes and generated `release.yml`.
2. Add `CHANGELOG.md` with a `## Unreleased` section.
3. Create `relateby/homebrew-tap` repo with a `Formula/README.md` stub.
4. Create fine-grained PAT for `homebrew-tap` with Contents write permission; add as `HOMEBREW_TAP_TOKEN` secret.
5. Create crates.io API token; add as `CARGO_REGISTRY_TOKEN` secret.
6. Add `cargo publish` step to `release.yml` (after GitHub Release creation, stable tags only).
7. Test with a `v0.1.0-beta.1` prerelease tag; verify GitHub Release is created, Homebrew formula is NOT updated, crates.io is NOT published.
8. Test with `v0.1.0` stable tag; verify all three channels update correctly.

**Validation**: `cargo dist build --target x86_64-unknown-linux-gnu` succeeds locally. All 5 archives + checksums appear in the GitHub Release.

---

### Phase 2 — npm distribution

**Goal**: `npm i -g @relateby/cli` installs the correct binary on all 5 platforms without any postinstall scripts.

**Tasks**:
1. Create `distribution/npm/platforms.tsv` with the 5-row platform mapping.
2. Create `distribution/npm/cli-platform/package.json.tmpl` for platform packages.
3. Create `distribution/npm/cli/package.json.tmpl` for the wrapper (with `optionalDependencies`).
4. Write `distribution/npm/cli/bin/relate.js` (the Node.js shim).
5. Write `publish-npm.yml` workflow:
   - Trigger: `release` event (type: `published`)
   - Steps: download release archives from GitHub Release → extract binaries → stamp `package.json` templates with release version → `npm publish` for all 6 packages
   - Use `NPM_TOKEN` secret; publish platform packages before the wrapper
   - Publish with `--tag latest` for stable, `--tag next` for prerelease
6. Create `@relateby` npm org; generate automation token; add as `NPM_TOKEN` secret.
7. Publish `0.0.0` placeholder packages to claim the 6 package names.
8. Test on macOS, Linux (x86_64), and Linux (arm64) with `--ignore-scripts`.

**Validation**: `npm i -g @relateby/cli && relate --version` works on all three tested platforms; only one platform binary package is downloaded per install.

---

### Phase 3 — Project website

**Goal**: `cli.relateby.dev` is live, OS-aware, and stays in sync with every stable release.

**Tasks**:
1. Build `website/index.html` — vanilla HTML/CSS/JS, no framework:
   - Hero section with OS-detecting install tabs (macOS · Linux · Windows · Homebrew · npm · cargo)
   - Copy-to-clipboard buttons
   - Quickstart section (4 steps)
   - Commands section with tabbed examples for `lint`, `parse`, `query`, `query --apply`, `mcp`
   - Agent Skills section
   - Full install matrix table
   - Light/dark theme toggle via CSS custom properties
   - Version badge (`<span id="current-version">`) updated by `update-website.yml`
2. Add self-hosted fonts to `website/fonts/`: Public Sans (prose), Fira Code (terminal).
3. Add `website/CNAME` containing `cli.relateby.dev`.
4. Add `website/robots.txt` and `website/sitemap.xml`.
5. Write `update-website.yml` workflow:
   - Trigger: `release` event (type: `published`, stable only — skip if tag has prerelease suffix)
   - Steps: checkout `gh-pages` → copy `install.sh`/`install.ps1` from release assets → update `<span id="current-version">` in `index.html` → commit and push to `gh-pages`
6. Configure DNS in GoDaddy: CNAME `cli` → `relateby.github.io`.
7. Enable GitHub Pages on `relateby/relate-cli` from `gh-pages` branch; set custom domain `cli.relateby.dev`; enable Enforce HTTPS after DNS propagates (~15–30 min).
8. Verify: `curl -sSfL https://cli.relateby.dev/install.sh | bash` succeeds on fresh macOS and Linux; website version matches latest GitHub Release.

**Validation**: Website loads at `https://cli.relateby.dev`; macOS tab is pre-selected on macOS; copy button works; mobile (375px) is readable.

## Risk Notes

- **cargo-dist version pinning**: The `cargo-dist-version` in `Cargo.toml` must be bumped when upgrading cargo-dist, otherwise CI uses a different version than was tested locally.
- **npm package name squatting**: The `@relateby` scope and all 6 package names must be claimed before any automated publish runs.
- **Homebrew PAT expiry**: The fine-grained PAT expires after 1 year. Set a calendar reminder to rotate it.
- **Tag/version mismatch**: If the git tag (`v0.2.0`) does not match `Cargo.toml` version (`0.2.0`), cargo-dist errors before publishing anything — desired behavior.
