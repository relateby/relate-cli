# Contract: Release Artifact Naming

This contract defines the stable naming conventions for all distribution artifacts.
Tooling (install scripts, npm shim, website) depends on these names being stable across releases.

## Version Tag Format

```
vMAJOR.MINOR.PATCH            # stable release
vMAJOR.MINOR.PATCH-LABEL.N   # prerelease (e.g., v0.2.0-beta.1, v0.3.0-rc.2)
```

The version in `Cargo.toml` MUST equal the tag without the leading `v`.

## GitHub Release Archive Names

```
relate-{TARGET}.tar.gz        # Linux and macOS
relate-{TARGET}.zip           # Windows
```

Where `{TARGET}` is the Rust target triple:
- `x86_64-unknown-linux-gnu`
- `aarch64-unknown-linux-gnu`
- `x86_64-apple-darwin`
- `aarch64-apple-darwin`
- `x86_64-pc-windows-msvc`

Example: `relate-x86_64-apple-darwin.tar.gz`

## Checksum File

```
relate-checksums.txt
```

One line per archive: `{sha256}  {filename}`. Used by `install.sh` for verification.

## dist-manifest

```
dist-manifest.json
```

Generated by cargo-dist. Machine-readable metadata consumed by the install scripts and any tooling that needs to enumerate release artifacts.

## Install Scripts

```
install.sh       # Unix (macOS + Linux)
install.ps1      # Windows PowerShell
```

These are served at `https://cli.relateby.dev/install.sh` and `https://cli.relateby.dev/install.ps1`. The content is generated by cargo-dist and MUST be updated with every release.

## npm Package Names

```
@relateby/cli                # wrapper (shim)
@relateby/cli-linux-x64
@relateby/cli-linux-arm64
@relateby/cli-darwin-x64
@relateby/cli-darwin-arm64
@relateby/cli-win32-x64
```

npm versions match the release version (without the leading `v`).

## Homebrew Formula

Formula file: `Formula/relate.rb` in `relateby/homebrew-tap`.
Install command: `brew install relateby/tap/relate`.
