# Implementation Plan: relate lint Command

**Branch**: `001-lint-command` | **Date**: 2026-05-10 | **Spec**: [spec.md](spec.md)

## Summary

Implement the `relate lint` subcommand stub in `src/commands/lint.rs` as a full linting
engine that dispatches `.cypher` and `.gram` files to the `cypher-data` and `gram-data`
lib targets respectively, extracts and lints code-fenced snippets from `.md` and `.adoc`
files, and renders diagnostics via `ariadne` (human-readable) or `serde_json` (JSON).
The shared `gram-diagnostics` type flows through unchanged; `relate` wraps it with
source context for output.

## Technical Context

**Language/Version**: Rust, edition 2021, rust-version 1.85.0  
**Primary Dependencies**:
- `cypher-data 0.2.3` — Cypher lint engine (lib target)
- `gram-data 0.3.10` — Gram lint engine (lib target)
- `gram-diagnostics 0.3.10` — shared `Diagnostic` type (serde built-in)
- `ariadne 0.6` — human-readable diagnostic rendering
- `walkdir 2` — recursive directory traversal
- `regex 1` — Markdown/AsciiDoc fence extraction
- `serde 1` + `serde_json 1` — JSON output
- Existing: `clap 4`, `anyhow 1`, `tokio 1` (full)

**Storage**: None (stateless CLI; reads files, writes stdout/stderr)  
**Testing**: `cargo test`; integration tests via `assert_cmd` + `predicates` in `tests/cli_integration.rs`  
**Target Platform**: macOS/Linux CLI binary  
**Project Type**: CLI (single-crate, no workspace)  
**Performance Goals**: Single file ≤2s; 500-file directory ≤30s  
**Constraints**: No Neo4j connection required for lint; sync command (no `.await`)  
**Scale/Scope**: File-at-a-time processing; no parallelism in v1

## Implementation Phases (for /speckit-tasks)

### Step 1 — Cargo.toml dependencies

Add to `[dependencies]`:
```toml
cypher-data = "0.2.3"
gram-data = "0.3.10"
gram-diagnostics = "0.3.10"
ariadne = "0.6"
walkdir = "2"
regex = "1"
serde = { version = "1", features = ["derive"] }
serde_json = "1"
```

Verify `cargo build` still passes.

### Step 2 — cli.rs: add Lang enum and --lang flag

In `src/cli.rs`:
- Add `Lang` enum implementing `clap::ValueEnum` with variants `Cypher` and `Gram`
- Add `lang: Lang` field to `LintArgs` with `default_value_t = Lang::Cypher`

Verify `relate lint --help` shows `--lang`.

### Step 3 — Internal types

In `src/commands/lint.rs`, define:
- `LintDiagnostic { lang: Lang, source_file: Option<PathBuf>, inner: gram_diagnostics::Diagnostic }`
- `Snippet { lang: Lang, source: String, fence_start_line: u32 }`
- `JsonDiagnostic` / `JsonRange` / `JsonPosition` serialization structs (rename `character` → `column`)

### Step 4 — Fence extraction

In `src/commands/lint.rs` (or inline `mod fence`):
- `extract_snippets(source: &str, path: &Path) -> Vec<Snippet>`
- Compile Markdown regex: `` /(?ms)^```[ \t]*(cypher|openCypher|gram)[ \t]*\n(.*?)^```[ \t]*$/ ``
- Compile AsciiDoc regex: `/(?ms)^\[source,[ \t]*(cypher|openCypher|gram)\]\n----\n(.*?)\n----/`
- Compute `fence_start_line` by counting `\n` chars before match start

Unit-test with a Markdown string containing one cypher fence and one gram fence.

### Step 5 — Dispatch and offset logic

`lint_path(path: &Path, opts: &LintOptions) -> anyhow::Result<Vec<LintDiagnostic>>`:
- `.cypher` → `cypher_data::lint::lint_file(path, opts)`
- `.gram` → `gram_data::lint::lint_file(path, opts)`
- `.md` / `.adoc` → read source → `extract_snippets` → per snippet: `lint_source` → offset lines → collect
- other → `anyhow::bail!("unsupported file type: {}", path.display())`

`offset_diagnostic(d: gram_diagnostics::Diagnostic, offset: u32) -> gram_diagnostics::Diagnostic`:
- Adds `offset` to `range.start.line` and `range.end.line`

### Step 6 — Directory walk

`collect_paths(inputs: &[PathBuf]) -> anyhow::Result<Vec<PathBuf>>`:
- For each input: if file → push directly (error on unsupported ext); if dir → WalkDir, filter for supported extensions, push

### Step 7 — Output rendering

`print_human(diagnostics: &[LintDiagnostic], sources: &HashMap<PathBuf, String>)`:
- Group by `source_file`
- For each file group: build ariadne `Report`, attach `Label` for each diagnostic, print via `Source::from(&source_str)`
- Convert `(line, character)` to byte offset for ariadne span: iterate source lines

`print_json(diagnostics: &[LintDiagnostic])`:
- Map each `LintDiagnostic` to `JsonDiagnostic`, serialize with `serde_json::to_string_pretty`

### Step 8 — Entry point (run function)

Wire `run(args: LintArgs) -> Result<()>`:
1. If `args.expr`: lint inline expression with selected lang → collect diagnostics
2. Else: `collect_paths` → for each path: `lint_path` → accumulate diagnostics
3. If `args.strict`: treat warnings as errors for exit code
4. Render: `print_json` or `print_human`
5. Return `Ok(())` (exit 0) or `std::process::exit(1)` on lint findings; `?` propagates tool errors (exit 2 via anyhow)

### Step 9 — Integration tests

In `tests/cli_integration.rs`, add tests covering:
- Clean `.cypher` file → exit 0
- `.cypher` file with violation → exit 1, stdout contains file:line:col
- `--json` on clean file → `[]`
- `--json` on file with violation → valid JSON with expected fields
- `--strict` with warning-only file → exit 1
- Directory walk with mixed file types → both cypher and gram diagnostics reported
- `.md` file with fenced cypher → diagnostic line references parent doc
- `--expr "MATCH (n) RETURN n"` → exit 0
- Unsupported file type passed explicitly → exit 2
